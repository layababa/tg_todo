## 💻 前端技术要求文档：Telegram To-Do Mini App

### 1. 核心技术栈与工具

| 领域 | 技术/库 | 具体要求 | 备注 |
| :--- | :--- | :--- | :--- |
| **框架** | **Vue 3** (Composition API) | 必须使用 Vue 3 进行开发，并推荐采用 Composition API 实现逻辑组织和代码复用。 | |
| **UI 库** | **daisyUI** | 必须使用 daisyUI 作为基础组件库。 | 基于 Tailwind CSS 的组件库，强调语义化类名和高度可定制性。 |
| **样式基础** | **Tailwind CSS** | 必须集成并配置 Tailwind CSS。 | 作为 daisyUI 的底层支持，用于快速构建和定制原子化样式。 |
| **动画库** | **uiverse.io** | 必须集成并使用来自 **uiverse.io** 的 CSS 动画。 | 用于增强关键交互和视觉效果，突出应用的时尚感。 |
| **状态管理** | Pinia | 推荐使用 Pinia 进行应用级状态管理。 | 适用于 Vue 3 的轻量级、类型安全的库。 |
| **构建工具** | Vite / Vue CLI | 推荐使用 Vite 以获得更快的开发体验和生产构建速度。 | 必须配置代码分割和 Tree Shaking 优化加载性能。 |

### 2. UI/UX 与样式定制要求

#### 2.1 主题与定制化
* **本地主题色集成**：UI 的 CSS 主题色必须从**./daisyUI.template.css**中读取和应用。
    * **实现方式**：利用 Tailwind 和 daisyUI 的主题配置能力，通过自定义配置文件将本地 CSS 文件中的颜色变量（CSS Variables）映射并注入到 daisyUI 的主题系统中。
* **非原生风格**：Mini App **不得**采用 Telegram 客户端的默认 UI 风格，而是采用 daisyUI 和自定义主题色定义的**时尚现代**风格。

#### 2.2 Telegram 集成与性能
* **TWA SDK**：必须集成 **Telegram Web App SDK (TWA SDK)**。
    * 用于用户身份认证（获取 `initData` 和 `User ID`）。
    * 用于控制 Mini App 的**主按钮 (`MainButton`)** 和**回退按钮**等原生 UI 元素。
* **性能优化**：Mini App 必须确保首屏加载速度快，打包体积小，以适应 Telegram WebView 的运行环境。

#### 2.3 动画集成
* **关键交互点**：必须在以下关键用户交互点集成 **uiverse.io** 上的动画，以提供独特的视觉反馈：
    * **待办完成标记**：使用 uiverse.io 动画增强复选框或切换按钮的选中/完成状态。
    * **主要操作按钮**：如列表页的删除、详情页的保存等操作，使用 uiverse.io 的按钮 hover/点击效果。
    * **加载状态**：在数据加载或 API 请求等待时，替换默认的加载图标，使用 uiverse.io 中具设计感的加载动画。
* **实现要求**：由于 uiverse.io 主要是纯 CSS 效果，前端实现时需确保这些自定义 CSS 类能够被局部应用到 Vue 组件中，不与 daisyUI 的基础样式产生大规模冲突。

### 3. 前端功能实现要求

| 功能点 | 实现细节 |
| :--- | :--- |
| **用户认证** | 必须在 Mini App 启动时，使用 TWA SDK 获取 `Web App Init Data` 并发送给后端进行安全校验和用户身份识别。 |
| **页面路由** | 使用 **Vue Router** 实现列表页和详情页之间的跳转，确保 SPA (Single Page Application) 体验。 |
| **后端通信** | 使用 **Axios** 或原生 `fetch` 封装统一的 API 服务，确保所有请求携带认证信息（如 `initData`）。 |
| **列表实现** | 使用 daisyUI 的 **Card/Table/List** 组件构建列表，并利用 Vue 的高性能渲染机制处理数据更新。 |
| **编辑交互** | 在待办详情/编辑页，使用 daisyUI 的 **`input`** 或 **`textarea`** 组件，实现标题内容的编辑和保存功能。 |
